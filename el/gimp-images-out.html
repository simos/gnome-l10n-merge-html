1. Αρχεία

Το GIMP είναι ικανό να διαβάζει και να γράφει μια μεγάλη ποικιλία μορφών αρχείων γραφικών. Με την εξαίρεση του εγγενούς τύπου αρχείου XCF του GIMP, ο χειρισμός αρχείου γίνεται από πρόσθετα. Έτσι είναι σχετικά εύκολο να επεκταθεί το GIMP για να υποστηρίξει νέους τύπους αρχείων, όταν προκύψει ανάγκη.

Δεν είναι όλοι οι τύποι αρχείων εξίσου καλοί για όλους τους σκοπούς. Αυτό το τμήμα της τεκμηρίωσης θα πρέπει να σας βοηθήσει να καταλάβετε τα πλεονεκτήματα και τα μειονεκτήματα κάθε τύπου.

1.1. Αποθήκευση εικόνων

Όταν τελειώσετε δουλεύοντας με μια εικόνα, θα θέλετε να αποθηκεύσετε τα αποτελέσματα. (Στην πραγματικότητα είναι συχνά καλή ιδέα να την αποθηκεύεται σε ενδιάμεσα στάδια επίσης.) Το GIMP είναι ένα πολύ αξιόπιστο πρόγραμμα, αλλά ακούσαμε φήμες, ίσως απόκρυφες, ότι είναι ίσως γνωστό σε περίεργες και μυστηριώδεις περιπτώσεις να καταρρέει. Οι περισσότερες μορφές αρχείου που το GIMP μπορεί να ανοίξει, μπορούν επίσης να αποθηκευτούν. Υπάρχει, όμως, μια μορφή αρχείου που είναι ειδική, η εγγενής μορφή XCF του GIMP και είναι χρήσιμη επειδή αποθηκεύει σχεδόν οτιδήποτε για μια εικόνα, (δεν αποθηκεύει πληροφορίες “αναίρεσης”). Γι' αυτό η μορφή XCF είναι ειδικά κατάλληλη για αποθήκευση ενδιάμεσων αποτελεσμάτων και για αποθήκευση εικόνων που θα ξανανοιχτούν αργότερα στο GIMP. Τα αρχεία XCF δεν διαβάζονται από τα περισσότερα άλλα προγράμματα που εμφανίζουν εικόνες. Έτσι, τελειώνοντας θα θέλετε προφανώς να αποθηκεύσετε την εικόνα σε μια πιο πλατιά χρησιμοποιούμενη μορφή όπως JPEG, PNG, TIFF, κλ.

1.2. Αποθήκευση αρχείων

Υπάρχουν αρκετές εντολές για αποθήκευση εικόνων. Ένας κατάλογος εντολών και πληροφορίες για τη χρήση τους, μπορείτε να βρείτε στην ενότητα που καλύπτει το Μενού αρχείο.

Το GIMP σας επιτρέπει να αποθηκεύσετε τις εικόνες που δημιουργείτε σε μια μεγάλη ποικιλία μορφών. Είναι σημαντικό να καταλάβετε ότι η μόνη μορφή ικανή να αποθηκεύσει όλες τις πληροφορίες σε μια εικόνα, που περιλαμβάνει στρώσεις, διαφάνειες κλ. είναι η εγγενής μορφή του GIMP XCF. Κάθε άλλη μορφή διατηρεί μερικές ιδιότητες της εικόνας και χάνει άλλες. Όταν αποθηκεύεται μια εικόνα, το GIMP προσπαθεί να σας προειδοποιήσει, αλλά εξαρτάται από εσάς να καταλάβετε τις δυνατότητες της μορφής που επιλέγετε.

Εικόνα 6.1. Παράδειγμα διαλόγου εξαγωγής


Όπως υπογραμμίστηκε πιο πάνω, δεν υπάρχει καμιά μορφή αρχείου, με την εξαίρεση της εγγενούς μορφής αρχείου του GIMP XCF, που είναι ικανή να αποθηκεύσει όλες τις πληροφορίες σε μια εικόνα του GIMP. Όταν ζητάτε να αποθηκευτεί μια εικόνα σε μορφή που θα χάσει πληροφορίες, το GIMP σας ενημερώνει, λέγοντας σας ποιές πληροφορίες θα χαθούν και σας ζητά εάν θα θέλατε να “εξαγάγετε” την εικόνα σε μορφή που ο τύπος αρχείου μπορεί να χειριστεί. Εξαγωγή μιας εικόνας δεν τροποποιεί την ίδια την εικόνα, έτσι δεν χάνετε τίποτα εξάγοντας. Δείτε Εξαγωγή αρχείου.

Σημείωση

Όταν κλείνετε μια εικόνα (πιθανόν αφήνοντας το GIMP), προειδοποιείστε εάν η εικόνα είναι "βρόμικη", δηλαδή εάν έχει αλλάξει χωρίς να έχει αποθηκευτεί. Αποθήκευση εικόνας σε οποιαδήποτε μορφή αρχείου θα σημάνει ότι θεωρείται "όχι βρόμικη", ακόμα κι αν η μορφή αρχείου δεν παρουσιάζει όλες τις πληροφορίες από την εικόνα.

1.2.1. Αποθήκευση ως GIF

Εικόνα 6.2. Ο διάλογος αποθήκευση ως GIF


Προειδοποίηση

Η μορφή αρχείου GIF δεν υποστηρίζει μερικές βασικές ιδιότητες της εικόνας, όπως ανάλυση εκτύπωσης. Εάν ενδιαφέρεσθε για αυτές τις ιδιότητες, χρησιμοποιείστε μια διαφορετική μορφή αρχείου όπως PNG.

GIF Options (Επιλογές GIF)
Interlace (Πεπλεγμένο)

Επιλέγοντας πεπλεγμένο επιτρέπεται σε μια εικόνα στον ιστό να εμφανιστεί σταδιακά καθώς φορτώνεται. Η σταδιακή εμφάνιση της εικόνας είναι χρήσιμη με ταχύτητες χαμηλής σύνδεσης, επειδή μπορείτε να σταματήσετε μια εικόνα που δεν σας ενδιαφέρει. Πεπλεγμένο είναι μικρής χρήσης σήμερα λόγω των υψηλότερων ταχυτήτων σύνδεσης.

GIF comment (Σχόλιο GIF)

Τα σχόλια GIF υποστηρίζουν μόνο 7-bit χαρακτήρες ASCII. Εάν χρησιμοποιήσετε χαρακτήρα εκτός του 7-bit συνόλου ASCII, το GIMP θα αποθηκεύσει την εικόνα χωρίς σχόλιο και θα σας ενημερώσει ότι το σχόλιο δεν αποθηκεύτηκε.

Animated GIF options (Επιλογές κινούμενου GIF)
Loop forever (Συνεχής βρόγχος)

Όταν επιλεγεί αυτή η επιλογή, η κίνηση θα επαναλαμβάνεται συνέχεια μέχρι να τη σταματήσετε.

Delay between frames if unspecified (καθυστέρηση μεταξύ πλαισίων (καρέ) εάν δεν ορίζεται)

Μπορείτε να βάλετε την καθυστέρηση μεταξύ πλαισίων(καρέ), σε χιλιοστά του δευτερολέπτου, εάν δεν έχει οριστεί πριν. Σ' αυτήν την περίπτωση, μπορείτε να τροποποιήσετε κάθε καθυστέρηση στο διάλογο στρώσης.

Frame disposal when unspecified (διευθέτηση πλαισίου(καρέ) όταν δεν προσδιορίζεται)

Εάν αυτό δεν έχει καθοριστεί, μπορείτε να ορίσετε πόσα πλαίσια (καρέ) θα υπερθέτονται. Μπορείτε να επιλέξετε μεταξύ τριών επιλογών:

  • I don't care (Δεν με ενδιαφέρει): Μπορείτε να χρησιμοποιήσετε αυτήν την επιλογή εάν όλες οι στρώσεις είναι αδιαφανείς. Οι στρώσεις θα αντικαταστήσουν ό,τι είναι από κάτω.

  • Cumulative Layers (combine) (Προσθετικές στρώσεις (συνδυασμός)): Τα προηγούμενα πλαίσια (καρέ) δεν θα καταργηθούν όταν εμφανίζεται ένα καινούργιο.

  • One frame per layer (replace) (Ένα πλαίσιο (καρέ) ανά στρώση (αντικατάσταση)): Τα προηγούμενα πλαίσια (καρέ) θα καταργηθούν πριν την εμφάνιση ενός καινούργιου.

1.2.2. Αποθήκευση ως JPEG

Τα αρχεία JPEG έχουν συνήθως μια επέκταση .jpg, .JPG, ή .jpeg. Είναι μια πολύ πλατιά χρησιμοποιούμενη μορφή, επειδή συμπιέζει τις εικόνες πολύ αποτελεσματικά, ενώ ελαχιστοποιεί την απώλεια ποιότητας της εικόνας. Καμιά άλλη μορφή δεν πλησιάζει στην επίτευξη του ίδιου επιπέδου συμπίεσης. Δεν υποστηρίζει, όμως, διαφάνεια ή πολλαπλές στρώσεις. Γι' αυτό, η αποθήκευση εικόνων ως JPEG συχνά απαιτεί να εξαχθούν από το GIMP.

Εικόνα 6.3. Διάλογος αποθήκευση ως JPEG


Ο αλγόριθμος του JPEG είναι περίπλοκος και εμπεριέχει έναν αριθμό επιλογών που προκαλούν σύγχυση και των οποίων το νόημα είναι πέρα από το σκοπό αυτής της τεκμηρίωσης. Εκτός και είσαστε ένας ειδικός του JPEG, η παράμετρος της ποιότητας είναι προφανώς η μόνη που θα χρειαστείτε να ρυθμίσετε.

Προσοχή

Αφού αποθηκεύσετε μια εικόνα ως αρχείο JPEG, η εικόνα δεν θεωρείται πια, “βρόμικη” από το GIMP, έτσι εκτός και κάνετε παραπέρα αλλαγές, δεν θα δεχθείτε καμία προειδοποίηση, εάν την κλείσετε. Επειδή η JPEG είναι με απώλειες και δεν υποστηρίζει διαφάνεια ή πολλαπλές στρώσεις , μερικές από τις πληροφορίες στην εικόνα μπορεί να χαθούν. Εάν θέλετε να αποθηκεύσετε όλες τις πληροφορίες σε μια εικόνα, χρησιμοποιείστε την εγγενή μορφή του GIMP XCF.

Ποιότητα

Όταν αποθηκεύετε ένα αρχείο σε μορφή JPEG, εμφανίζεται ένας διάλογος που σας επιτρέπει να ορίσετε το επίπεδο ποιότητας, που κυμαίνεται από 0 μέχρι 100. Τιμές πάνω από 95 δεν είναι γενικά χρήσιμες. Η προκαθορισμένη ποιότητα του 85 συνήθως παράγει θαυμάσια αποτελέσματα, αλλά σε πολλές περιπτώσεις είναι δυνατό να οριστεί η ποιότητα πραγματικά χαμηλότερη χωρίς υποβάθμιση της εικόνας. Μπορείτε να ελέγξετε το αποτέλεσμα ρυθμίσεων διαφορετικής ποιότητας επιλέγοντας Show Preview in image window (Προβολή προεπισκόπησης στο παράθυρο εικόνας) στο διάλογο JPEG.

Σημείωση

Σημειώστε ότι οι αριθμοί για το επίπεδο ποιότητας JPEG έχουν διαφορετικό νόημα σε διάφορες εφαρμογές. Αποθήκευση με επίπεδο ποιότητας 80 στο GIMP δεν είναι κατ' ανάγκη συγκρίσιμο με αποθήκευση με επίπεδο ποιότητας 80 σε άλλη εφαρμογή.

Preview in image window (Προεπισκόπηση στο παράθυρο εικόνας)

Σημειώνοντας αυτήν την επιλογή, κάθε αλλαγή στην ποιότητα (ή άλλη παράμετρο του JPEG) παρουσιάζεται στην εμφάνιση εικόνας. (Αυτό δεν αλλάζει την εικόνα. Η εικόνα ξαναμετατρέπεται στην αρχική της κατάσταση, όταν ο διάλογος JPEG κλείνει.)

Advanced settings (Προχωρημένες ρυθμίσεις)

Μερικές πληροφορίες για τις προχωρημένες ρυθμίσεις.

Optimize (Βελτιστοποίηση)

Εάν ενεργοποιήσετε αυτήν την επιλογή, η βελτιστοποίηση των παραμέτρων για κωδικοποίηση της εντροπίας θα χρησιμοποιηθεί. Το αποτέλεσμα είναι γενικά ένα πιο μικρό αρχείο, αλλά παίρνει πιο πολύ χρόνο να δημιουργηθεί.

Progressive (Σταδιακό)

Με αυτήν την επιλογή επισημασμένη, τα κομμάτια της εικόνας αποθηκεύονται σε ένα αρχείο με σειρά που επιτρέπει σταδιακή βελτιστοποίηση της εικόνας κατά τη διάρκεια αργής σύνδεσης με το διαδίκτυο. Η σταδιακή επιλογή για JPG έχει τον ίδιο σκοπό όπως η πεπλεγμένη επιλογή για GIF. Δυστυχώς, η σταδιακή επιλογή παράγει ελαφρώς μεγαλύτερα αρχεία JPG.

Save EXIF data (Αποθήκευση δεδομένων EXIF)

Τα αρχεία JPEG από πολλές ψηφιακές κάμερες περιέχουν επιπρόσθετες πληροφορίες, που λέγονται δεδομένα EXIF. Δεδομένα EXIF παρέχουν πληροφορίες για την εικόνα, όπως κατασκευαστή και μοντέλο, μέγεθος εικόνας, ημερομηνία εικόνας κλ. Αν και το GIMP χρησιμοποιεί τη βιβλιοθήκη “libexif” για ανάγνωση και εγγραφή δεδομένων EXIF, η βιβλιοθήκη δεν πακετάρεται αυτόματα με το GIMP. Εάν το GIMP δημιουργήθηκε με libexif υποστήριξη, τότε τα δεδομένα EXIF διατηρούνται εάν ανοίξετε ένα αρχείο JPEG, το επεξεργαστείτε και έπειτα το αποθηκεύσετε ως JPEG. Τα δεδομένα EXIF δεν αλλάζουν καθόλου, εάν το κάνετε αυτό. Τα δεδομένα EXIF μπορεί να δείχνουν πράγματα όπως χρόνος δημιουργίας της εικόνας και όνομα αρχείου, που μπορεί να μην είναι πια σωστό. Εάν το GIMP δεν δημιουργήθηκε με υποστήριξη EXIF, μπορείτε να ανοίξετε αρχεία JPG που περιέχουν δεδομένα EXIF, αλλά τα δεδομένα EXIF παραλείπονται και δεν αποθηκεύονται όταν η τελική εικόνα αποθηκευτεί αργότερα.

Save thumbnail (Αποθήκευση μικρογραφίας)

Αυτή η επιλογή σας επιτρέπει να αποθηκεύσετε μια μικρογραφία με την εικόνα. Πολλές εφαρμογές χρησιμοποιούν την εικόνα μικρογραφίας ως μια γρήγορη μικρή διαθέσιμη προεπισκόπηση εικόνας.

Σημείωση

Αυτή η επιλογή εμφανίζεται μόνο αν το GIMP δημιουργήθηκε με υποστήριξη EXIF.

Save XMP data (Αποθήκευση δεδομένων XMP)

Τα δεδομένα XMP είναι “meta” δεδομένα για την εικόνα. Είναι μια ανταγωνιστική μορφή με το EXIF. Εάν ενεργοποιήσετε αυτήν την επιλογή, τα meta δεδομένα της εικόνας αποθηκεύονται σε μια δομή XMP μες το αρχείο.

Use quality settings from original image (Χρήση ποιοτικών ρυθμίσεων από την αυθεντική εικόνα).

Εάν μία ειδική ποιοτική ρύθμιση (ή “πίνακας ποιοτικοποίησης ”) προσαρτήθηκε στην εικόνα όταν φορτώθηκε, τότε αυτή η επιλογή σας επιτρέπει να την χρησιμοποιήσετε αντί για τις προκαθορισμένες.

Εάν κάνατε μόνο λίγες αλλαγές στην εικόνα, έπειτα ξαναχρησιμοποιήσατε την ίδια ρύθμιση ποιότητας θα πάρετε σχεδόν την ίδια ποιότητα και μέγεθος αρχείου όπως και η αυθεντική. Αυτό θα ελαχιστοποιήσει τις απώλειες που προκλήθηκαν από το βήμα της ποιοτικοποίησης, συγκρινόμενο με το τι θα συνέβαινε εάν χρησιμοποιούσατε διαφορετική ρύθμιση ποιότητας.

Εάν οι ρυθμίσεις ποιότητας που βρίσκονται στο αρχικό αρχείο δεν είναι καλύτερες από τις προκαθορισμένες ποιοτικές ρυθμίσεις, τότε η επιλογή “Use quality settings from original image (Χρήση ποιοτικών ρυθμίσεων από την αυθεντική εικόνα)” θα είναι διαθέσιμες, αλλά όχι ενεργοποιημένες. Αυτό επιβεβαιώνει ότι θα παίρνετε πάντοτε τουλάχιστον την ελάχιστη ποιότητα που προδιαγράφεται από προεπιλογή. Εάν δεν κάνατε μεγάλες αλλαγές στην εικόνα και θέλετε να την αποθηκεύσετε χρησιμοποιώντας την ίδια ποιότητα όπως η αρχική, τότε μπορείτε να το κάνετε ενεργοποιώντας αυτήν την επιλογή.

Smoothing (Εξομάλυνση)

Η συμπίεση JPG δημιουργεί παράσιτα. Χρησιμοποιώντας αυτή την επιλογή, μπορείτε να εξομαλύνετε την εικόνα όταν αποθηκεύεται, μειώνοντας την. Αλλά η εικόνα σας γίνεται κάπως θολή.

Restart markers (Επανεκκίνηση σημειωτών)

Το αρχείο εικόνας μπορεί να περιέχει σημειωτές που επιτρέπουν στην εικόνα να φορτωθεί ως τμήματα. Εάν η σύνδεση διακοπεί, ενώ φορτώνεται η εικόνα σε μια ιστοσελίδα, η φόρτωση μπορεί να συνεχιστεί από τον επόμενο σημειωτή.

Subsampling (Υποδειγματοληψία)

Το ανθρώπινο μάτι δεν είναι ευαίσθητο το με τον ίδιο τρόπο σε όλο το χρωματικό φάσμα. Η συμπίεση μπορεί να το χρησιμοποιήσει για να τροποποιήσει ελαφρά διαφορετικά χρώματα, ως ίδια. Υπάρχουν τρεις μέθοδοι.

  • 1x1,1x1,1x1 (best quality) (άριστη ποιότητα): Συνήθως αναφερόμενο ως (4:4:4), παράγοντας την καλύτερη ποιότητα, διατηρώντας τα περιγράμματα και αντιθέσεις χρωμάτων. αλλά η συμπίεση είναι πιο μικρή.

  • 2x1,1x1,1x1 (4:2:2): Αυτή είναι η τυπική υποδειγματοληψία, που συνήθως προσφέρει καλή αναλογία μεταξύ ποιότητας εικόνας και μεγέθους αρχείου. Υπάρχουν, όμως, περιπτώσεις όπου χρησιμοποιώντας όχι υποδειγματοληψία (4:4:4) προσφέρει μια αξιοσημείωτη αύξηση στην ποιότητα της εικόνας. Για παράδειγμα, όταν η εικόνα περιέχει λεπτές λεπτομέρειες όπως κείμενο πάνω από ομοιόμορφο φόντο, ή φωτογραφίες με σχεδόν επίπεδα χρώματα.

  • 1x2,1x1,1x1 Αυτό είναι παρόμοιο με το (2x1,1x1,1x1), αλλά η δειγματοληψία χρώματος είναι στην οριζόντια κατεύθυνση παρά στην κάθετη κατεύθυνση, σαν κάποιος να περιέστρεψε την εικόνα.

  • 2x2,1x1,1x1 (smallest file) (ελάχιστο αρχείο) : Συνήθως αναφέρεται ως (4:1:1), παράγοντας ελάχιστα αρχεία. Αυτό ταιριάζει σε εικόνες με ασθενή περιγράμματα, αλλά τείνει να μετουσιώσει τα χρώματα.

DCT Method (Μέθοδος DCT)

DCT είναι “discrete cosine transform (προσεκτικός μετασχηματισμός συνημιτόνου)” και είναι το πρώτο βήμα στον αλγόριθμο JPEG πηγαίνοντας από το χωρικό στο συχνοτικό τομέα. Οι επιλογές είναι “float (κινητή υποδιαστολή)”, “integer (ακέραιος)” (η προεπιλεγμένη) και “fast integer (γρήγορος ακέραιος)”.

  • Η float (κινητή υποδιαστολή) . Η μέθοδος κινητής υποδιαστολής είναι ελαφρά πιο ακριβής από τη μέθοδο ακέραιου, αλλά είναι πολύ πιο αργή εκτός και το μηχάνημα σας έχει πολύ γρήγορο υλικό κινητής υποδιαστολής. Επίσης σημειώστε ότι τα αποτελέσματα της μεθόδου κινητής υποδιαστολής μπορεί να διαφέρουν ελαφρά μεταξύ των υπολογιστών, ενώ οι μέθοδοι ακέραιου πρέπει να δίνουν τα ίδια αποτελέσματα παντού.

  • integer (ακέραιος) (Η προεπιλεγμένη). Αυτή η μέθοδος είναι πιο γρήγορη από την “float (κινητή υποδιαστολή)”, αλλά όχι τόσο ακριβής.

  • fast integer (γρήγορος ακέραιος). Η μέθοδος γρήγορου ακέραιου είναι πολύ λιγότερο ακριβής από τις άλλες δύο.

Σχόλιο

Σε αυτό το πλαίσιο κειμένου μπορείτε να εισαγάγετε ένα σχόλιο που αποθηκεύεται μαζί με την εικόνα.

1.2.3. Αποθήκευση ως PNG

Εικόνα 6.4. Ο διάλογος “Αποθήκευση ως PNG”


Interlacing (πεπλεγμένο)

Έλεγχος της πλέξης επιτρέπει στην εικόνα σε μια ιστοσελίδα να εμφανίζεται σταδιακά καθώς λαμβάνεται. Σταδιακή εμφάνιση της εικόνας είναι χρήσιμη με ταχύτητες αργής σύνδεσης, επειδή μπορείτε να διακόψετε μια εικόνα που δεν σας ενδιαφέρει. Η πλέξη είναι μικρής χρήσης σήμερα με τις γρήγορες ταχύτητες σύνδεσης.

Save background color (Αποθήκευση χρώματος παρασκηνίου)

Εάν η εικόνα σας έχει πολλά επίπεδα διαφάνειας, οι περιηγητές διαδικτύου που αναγνωρίζουν μόνο δύο επίπεδα, θα χρησιμοποιήσουν το χρώμα φόντου της εργαλειοθήκης σας στη θέση τους. Ο Internet Explorer μέχρι την έκδοση 6 δεν χρησιμοποιεί αυτήν την πληροφορία.

Save gamma (Αποθήκευση γάμα)

Η διόρθωση γάμα είναι η ικανότητα να διορθώνονται οι διαφορές στην ερμηνεία τιμών χρωμάτων. Αυτό αποθηκεύει την πληροφορία γάμα στην PNG που αντανακλά τον τρέχοντα παράγοντα γάμα για την εμφάνιση σας. Θεατές σε άλλους υπολογιστές μπορούν τότε να αντισταθμίσουν για να επιβεβαιώσουν ότι η εικόνα δεν είναι ούτε πολύ σκοτεινή, ούτε that the image πολύ φωτεινή.

Save layer offset (Αποθήκευση μετατόπισης στρώσης)

Το PNG υποστηρίζει μια τιμή μετατόπισης που λέγεται “oFFs chunk”, που δίνει τα δεδομένα θέσης. Δυστυχώς, η υποστήριξη μετατόπισης PNG στο GIMP, είναι ελαττωματική, ή τουλάχιστον ασύμβατη με άλλες εφαρμογές και για πολύ χρόνο. Μην ενεργοποιείτε τις μετατοπίσεις, αφήστε το GIMP να ισοπεδώσει τις στρώσεις πριν την αποθήκευση και δεν θα έχετε προβλήματα.

Save Resolution (Αποθήκευση ανάλυσης)

Αποθήκευση της ανάλυσης εικόνας, σε DPI (κουκίδες ανά ίντσα).

Save creation time (Αποθήκευση χρόνου δημιουργίας)

Ημερομηνία αποθήκευσης του αρχείου.

Save comment (Αποθήκευση σχολίου)

Μπορείτε να διαβάσετε αυτό το σχόλιο Ιδιότητες εικόνας.

Save color values from transparent pixels (Αποθήκευση τιμών χρώματος από διαφανή εικονοστοιχεία)

Με ενεργοποιημένη αυτήν την επιλογή, οι τιμές χρώματος αποθηκεύονται ακόμα κι αν τα εικονοστοιχεία είναι ολότελα διαφανή.

Compression level (Επίπεδο συμπίεσης)

Επειδή η συμπίεση είναι χωρίς απώλειες, η μόνη αιτία για χρήση επιπέδου συμπίεσης πιο μικρό από 9, είναι εάν παίρνει πολύ χρόνο για συμπίεση ενός αρχείου σε αργό υπολογιστή. Κανένας φόβος από την αποσυμπίεση. Είναι γρήγορη ανεξάρτητα από το επίπεδο συμπίεσης.

Save Defaults (Αποθήκευση προκαθορισμένων)

Πατήστε για αποθήκευση των τρεχουσών ρυθμίσεων. Έπειτα, μπορείτε να χρησιμοποιήσετε Load Defaults (Φόρτωση προεπιλεγμένων) για να φορτώσετε τις αποθηκευμένες ρυθμίσεις.

Σημείωση

Η μορφή PNG υποστηρίζει εικόνες από χρωματολόγιο. Χρησιμοποιώντας λιγότερα χρώματα, άρα, καταλήγουμε σε πιο μικρά αρχεία. Αυτό είναι ιδιαίτερα χρήσιμο για δημιουργία εικόνων στον ιστό. Δείτε Ενότητα 6.6, “Κατάσταση από χρωματολόγιο”.

Οι υπολογιστές εργάζονται σε μπλοκ των 8 bits που ονομάζονται “Byte”. Ένα byte επιτρέπει 256 χρώματα. Μειώνοντας τον αριθμό των χρωμάτων κάτω από 256 δεν είναι χρήσιμο. Ένα byte θα χρησιμοποιηθεί οπωσδήποτε και το μέγεθος του αρχείου δεν θα μικρύνει. Επιπρόσθετα, αυτή η “PNG8” μορφή, όπως το GIF, χρησιμοποιεί μόνο ένα bit για διαφάνεια. Μόνο δύο επίπεδα διαφάνειας είναι δυνατά, διαφανές ή αδιαφανές.

Εάν θέλετε η διαφάνεια PNG να εμφανίζεται πλήρως από τον Internet Explorer, μπορείτε να χρησιμοποιήσετε το φίλτρο AlphaImageLoader DirectX στον κώδικα της ιστοσελίδας σας. Δείτε Microsoft Knowledge Base [MSKB-294714]. Παρακαλώ σημειώστε, ότι αυτό δεν είναι κατ' ανάγκη για τον Internet Explorer 7 και πάνω.

Συμπίεση

Αυτή η επιλογή σας επιτρέπει να ορίσετε το χρησιμοποιούμενο αλγόριθμο για συμπίεση της εικόνας.

  • None (Κανένας): Είναι γρήγορος και χωρίς απώλειες, αλλά το προκύπτον αρχείο είναι πολύ μεγάλο.

  • LZW. Η εικόνα συμπιέζεται χρησιμοποιώντας τον αλγόριθμο “Lempel-Ziv-Welch”, μια τεχνική συμπίεσης χωρίς απώλειες. Αυτός είναι παλιός, αλλά αποτελεσματικός και γρήγορος. Περισσότερες πληροφορίες στο: [WKPD-LZW].

  • Pack Bits (Bits ανά πακέτο): Γρήγορο, απλό σχήμα συμπίεσης για κωδικοποίηση δεδομένων. Η Apple εισήγαγε την μορφή PackBits με την έκδοση του MacPaint σε υπολογιστή Macintosh. Μια ροή δεδομένων PackBits αποτελείται από πακέτα επικεφαλίδας του ενός byte ακολουθούμενα από δεδομένα. (Πηγή: [WKPD-PACKBITS])

  • Deflate (Ξεφούσκωμα): Είναι ένας αλγόριθμος συμπίεσης δεδομένων χωρίς απώλειες που χρησιμοποιεί ένα συνδυασμό του αλγόριθμου LZ77 και του κώδικα Huffman. Χρησιμοποιείται επίσης σε μορφές αρχείων Zip, Gzip και PNG. Πηγή:: [WKPD-DEFLATE].

  • JPEG: Είναι ένας πολύ καλός αλγόριθμος συμπίεσης, αλλά με απώλειες.

  • CCITT Group 3 fax; CCITT Group 4 fax είναι μια μορφή ασπρόμαυρη που αναπτύχθηκε για μεταφορά εικόνων με φαξ.

    Σημείωση

    Αυτές οι επιλογές μπορεί να επιλεγούν μόνο εάν η εικόνα είναι σε κατάσταση χρωματολογίου και μειωθεί σε δύο χρώματα. Πατήστε Εικόνα → Κατάσταση → Από χρωματολόγιο για μετατροπή σε εικόνα από χρωματολόγιο. Βεβαιωθείτε για την ενεργοποίηση “Χρήση ασπρόμαυρης (1-bit)παλέτας”.

Save color values from transparent pixels (Αποθήκευση χρωματικών τιμών από διαφανή εικονοστοιχεία)

Με αυτήν την επιλογή οι χρωματικές τιμές αποθηκεύονται ακόμα κι αν τα εικονοστοιχεία είναι ολότελα διαφανή.
