3.4. Το πρώτο σας σενάριο Script-Fu

Ας προχωρήσουμε με το τέταρτο μάθημα, το πρώτο σας σενάριο Script-Fu.

3.4.1. Δημιουργία ενός πλαισίου κειμένου σεναρίου

Μία από τις πιο συχνές ενέργειες που εκτελώ στο GIMP είναι η δημιουργία ενός πλαισίου με κάποιο κείμενο σε αυτό για μια ιστοσελίδα, ένα λογότυπο ή οτιδήποτε. Όμως, δεν ξέρετε ποτέ απολύτως το μέγεθος της αρχικής εικόνας, όταν ξεκινάτε. Δεν ξέρετε πόσο χώρο το κείμενο θα γεμίσει με τη γραμματοσειρά και το μέγεθος της γραμματοσειράς.

Ο ειδικός στο Script-Fu Master (και ο σπουδαστής) θα καταλάβουν γρήγορα ότι αυτό το πρόβλημα μπορεί εύκολα να λυθεί και να αυτοματοποιηθεί με το Script-Fu.

Θα δημιουργήσουμε ένα σενάριο, αποκαλούμενο πλαίσιο κειμένου, που δημιουργεί μια εικόνα σε σωστό μέγεθος για να ταιριάσει άνετα γύρω από μια γραμμή κειμένου που εισάγει ο χρήστης. Επίσης θα αφήσουμε το χρήστη να επιλέξει τη γραμματοσειρά, το μέγεθος της και χρώμα κειμένου.

3.4.2. Επεξεργασία και αποθήκευση των σεναρίων σας

Μέχρι τώρα, δουλέψαμε στην κονσόλα Script-Fu. Τώρα, θα αλλάξουμε στην επεξεργασία αρχείων κειμένου του σεναρίου.

Που βάζετε τα σενάρια σας είναι θέμα επιλογής — εάν έχετε πρόσβαση στον προεπιλεγμένο κατάλογο σεναρίων του GIMP, μπορείτε να βάλετε τα σενάρια σας, εκεί. Όμως, προτιμώ να κρατώ τα προσωπικά μου σενάρια στο δικό μου κατάλογο σεναρίων, για να τα κρατώ ξέχωρα από τα εγκατεστημένα σενάρια.

Στον κατάλογο .gimp-2.6 που το GIMP δημιούργησε στον προσωπικό σας κατάλογο, θα πρέπει να βρείτε έναν κατάλογο που λέγεται scripts. Το GIMP θα κοιτάξει αυτόματα στον κατάλογο σας .gimp-2.6 για έναν κατάλογο scripts και θα προσθέσει τα σενάρια σε αυτόν τον κατάλογο στη βάση δεδομένων του Script-Fu. Θα πρέπει να βάλετε τα προσωπικά σας σενάρια εδώ.

3.4.3. Τα απολύτως αναγκαία

Κάθε σενάριο Script-Fu ορίζει τουλάχιστον μία συνάρτηση που είναι η κύρια συνάρτηση του σεναρίου. Εκεί πρέπει να δουλέψετε.

Κάθε σενάριο πρέπει επίσης να καταχωρείται στη διαδικασιακή βάση δεδομένων, ώστε να μπορείτε να την προσπελάσετε μες το GIMP.

Θα ορίσουμε καταρχήν την κύρια συνάρτηση πρώτα:

(define (script-fu-text-box inText inFont inFontSize inTextColor))

Εδώ, ορίσαμε μια νέα συνάρτηση που λέγεται script-fu-text-box που δέχεται τέσσερις παραμέτρους που θα αντιστοιχούν αργότερα σε: κάποιο κείμενο, μια γραμματοσειρά, το μέγεθος μιας γραμματοσειράς, και το χρώμα του κειμένου. Η συνάρτηση είναι προς το παρόν άδεια και δεν κάνει τίποτα.

3.4.4. Ονοματικές συμβάσεις

Οι ονοματικές συμβάσεις του Scheme φαίνεται να προτιμούν τα μικρά γράμματα με ενωτικές παύλες, που ακολούθησα στην ονομασία των συναρτήσεων. Όμως, απομακρύνθηκα από τη σύμβαση με τις παραμέτρους. Μ' αρέσουν πιο περιγραφικά ονόματα για τις παραμέτρους και τις μεταβλητές και έτσι προσέθεσα το πρόθεμα "in" στις παραμέτρους, για να μπορώ να βλέπω γρήγορα ότι είναι τιμές που πέρασαν στο σενάριο, παρά δημιουργήθηκαν μέσα σ' αυτό. Χρησιμοποίησα το πρόθεμα "the" για μεταβλητές οριζόμενες μέσα στο σενάριο.

Είναι σύμβαση του GIMP να ονομάζει τις συναρτήσεις του σεναρίου σας script-fu-abc, επειδή τότε όταν καταχωρούνται στην διαδικασιακή βάση δεδομένων, θα εμφανίζονται όλες κάτω από το script-fu, όταν καταχωρίζετε τις συναρτήσεις. Επίσης αυτό βοηθά στη διάκριση τους από τα πρόσθετα.

3.4.5. Καταχώρηση της συνάρτησης

Τώρα, ας καταχωρίσουμε τη συνάρτηση με το GIMP. Αυτό γίνεται καλώντας τη συνάρτηση.script-fu-register. Όταν το GIMP διαβάζει ένα σενάριο, θα εκτελεί αυτή τη συνάρτηση που καταχωρίζει το σενάριο με την διαδικασιακή βάση δεδομένων. Μπορείτε να καταχωρίσετε αυτήν την συναρτησιακή κλήση όπου επιθυμείτε στο σενάριο, αλλά συνήθως την τοποθετώ στο τέλος, μετά τον υπόλοιπο κώδικα.

Να μια λίστα για καταχώρηση αυτής της συνάρτησης (θα εξηγήσω όλες τις παραμέτρους σε λίγο): 

(script-fu-register
"script-fu-text-box"                        ;func name
"Text Box"                                  ;menu label
"Creates a simple text box, sized to fit\
around the user's choice of text,\
font, font size, and color."              ;description
"Michael Terry"                             ;author
"copyright 1997, Michael Terry;\
2009, the GIMP Documentation Team"        ;copyright notice
"October 27, 1997"                          ;date created
""                     ;image type that the script works on
SF-STRING      "Text"          "Text Box"   ;a string variable
SF-FONT        "Font"          "Charter"    ;a font variable
SF-ADJUSTMENT  "Font size"     '(50 1 1000 1 10 0 1)
;a spin-button
SF-COLOR       "Color"         '(0 0 0)     ;color variable
)
(script-fu-menu-register "script-fu-text-box" "<Image>/File/Create/Text")

Εάν αποθηκεύσετε αυτές τις συναρτήσεις σε αρχείο κειμένου με την κατάληξη .scm στον κατάλογο σεναρίων σας, τότε επιλέξτε Φίλτρα → Script-Fu → Ανανέωση σεναρίων, αυτό το νέο σενάριο θα εμφανιστεί ως Αρχείο → Δημιουργία → Text → Text Box.

Εάν καλέσετε αυτό το νέο σενάριο, δεν θα κάνει τίποτα, φυσικά, αλλά μπορείτε να δείτε τις προτροπές που δημιουργήσατε καταχωρίζοντας το σενάριο (περισσότερες πληροφορίες πιο κάτω).

Τελικά, εάν καλέσετε τον Procedure Browser (περιηγητής διαδικασίας)( Βοήθεια → Procedure Browser (περιηγητής διαδικασίας)), θα σημειώσετε ότι το σενάριο μας, εμφανίζεται τώρα στη βάση δεδομένων.

3.4.6. Βήματα για καταχώριση του σεναρίου

Για καταχώριση του σεναρίου μας με το GIMP, καλούμε τη συνάρτηση script-fu-register, συμπληρωμένη στις επτά απαραίτητες παραμέτρους προσθέτοντας τις δικές μας παραμέτρους του σεναρίου μαζί με μια περιγραφή και προεπιλεγμένες τιμές για κάθε παράμετρο.

Οι απαιτούμενες παράμετροι

  • Το όνομα της συνάρτησης που ορίσαμε. Αυτή είναι η καλούμενη συνάρτηση όταν το σενάριο μας καλείται (το σημείο εισόδου στο σενάριο μας). Αυτό είναι απαραίτητο επειδή μπορεί να ορίσουμε πρόσθετες συναρτήσεις μες το ίδιο αρχείο και το GIMP χρειάζεται να ξέρει ποια από αυτές τις συναρτήσεις καλούμε. Στο παράδειγμα μας, ορίζουμε μόνο μια συνάρτηση, text-box που καταχωρίζεται.

  • Η τοποθεσία στο μενού, όπου θα εισαχθεί το σενάριο. Η ακριβής τοποθεσία του σεναρίου ορίζεται ως διαδρομή στο Unix, με τη ρίζα της διαδρομής να είναι μενού εικόνας όπως <Image>.[5]

    Εάν το σενάριο σας δεν λειτουργεί σε μια υπάρχουσα εικόνα (και έτσι δημιουργεί μια νέα εικόνα, όπως το σενάριο του Text Box κάνει), θα θέλετε να το εισαγάγετε στο μενού παραθύρου της εικόνας που μπορείτε να προσπελάσετε μέσα από τη γραμμή μενού της εικόνας, δεξιοπατώντας το παράθυρο εικόνας, πατώντας το εικονίδιο του κουμπιού του μενού στην πάνω αριστερή γωνία του παραθύρου εικόνας, ή πατώντας F10.

    Εάν το σενάριο σας προορίζεται να δουλέψει σε μια επεξεργασμένη εικόνα, θα θέλετε να το εισαγάγετε στο μενού του εικονοπαραθύρου. Το υπόλοιπο της διαδρομής δείχνει στις λίστες μενού, στα μενού και τα υπομενού. Έτσι, καταχωρίσαμε το σενάριο Text Box στο μενού (Αρχείο → Δημιουργία → Text → Text Box).

    Εάν παρατηρήσατε, το υπομενού κειμένου στο μενού Αρχείο/Δημιουργία δεν υπήρχε όταν ξεκινήσαμε — το GIMP αυτόματα δημιουργεί κάθε μενού που δεν υπάρχει ήδη.

  • Μια περιγραφή του σεναρίου που θα εμφανιστεί στο Procedure Browser (περιηγητή διαδικασίας).

  • Το όνομά σας (Ο συγγραφέας του σεναρίου).

  • Πληροφορίες για το Copyright.

  • Η ημερομηνία δημιουργίας του σεναρίου, ή η τελευταία αναθεώρηση του σεναρίου.

  • Οι τύποι των εικόνων που δουλεύει το σενάριο. Αυτό μπορεί να είναι οποιοδήποτε από τα παρακάτω: RGB, RGBA, GRAY, GRAYA, INDEXED, INDEXEDA. Ή μπορεί να μην είναι κανένα — στην περίπτωση μας δημιουργούμε μια εικόνα και έτσι δεν χρειάζεται να ορίσουμε τον τύπο της εικόνας που δουλεύουμε.

Εικόνα 12.4. Το μενού του σεναρίου μας.

SF-FONT

Δημιουργεί μια εφαρμογή επιλογής γραμματοσειράς στο διάλογο. Επιστρέφει ένα όνομα γραμματοσειράς ως αλφαριθμητικό. Υπάρχουν δύο νέες διαδικασίες κειμένου gimp για διευκόλυνση της χρήσης αυτής της επιστρεφόμενης παραμέτρου:

(gimp-text-fontname image drawable x-pos y-pos text border antialias size unit font)

(gimp-text-get-extents-fontname text size unit font)

όπου font είναι το όνομα της γραμματοσειράς που παίρνετε. Το οριζόμενο μέγεθος στο fontname αγνοείται σιωπηρά. Χρησιμοποιείται μόνο στον επιλογέα γραμματοσειράς. Σας ζητιέται να ορίσετε μια χρήσιμη τιμή (24 εικονοστοιχεία είναι μια καλή επιλογή).

SF-FONT "label" "fontname"

"label"Το τυπωμένο κείμενο πριν την ζώνη διαλόγου.

SF-BRUSH

Θα δημιουργήσει μια ζώνη επιλογής πινέλου. Η ζώνη επιλογής αποτελείται από μια περιοχή προεπισκόπησης (που όταν πατιέται θα παράξει μια αναδυόμενη προεπισκόπηση) και ένα κουμπί με την ετικέτα "...". Το κουμπί θα εμφανίσει ένα διάλογο, όπου πινέλα μπορούν να επιλεγούν και καθένα από τα χαρακτηριστικά του πινέλου μπορεί να τροποποιηθεί.

SF-BRUSH "Brush" '("Circle (03)" 100 44 0)

Εδώ ο διάλογος του πινέλου θα αναδυθεί με ένα προκαθορισμένο πινέλο κύκλος (03) αδιαφάνεια 100 διάκενο 44 και κανονική κατάσταση βαφής (τιμή 0).

Εάν αυτή η επιλογή δεν αλλάχτηκε η τιμή που περνά στη συνάρτηση ως παράμετρος θα ήταν '("κύκλος (03)" 100 44 0).

SF-PATTERN

Θα δημιουργήσει μια ζώνη επιλογής στο διάλογο ελέγχου που αποτελείται από μια περιοχή προεπισκόπησης (που όταν πατηθεί θα παράξει μια αναδυόμενη προεπισκόπηση) και ένα κουμπί με την ετικέτα "...". Το κουμπί θα εμφανίσει ένα διάλογο όπου μοτίβα μπορούν να επιλεγούν.

SF-PATTERN "Pattern" "Maple Leaves"

Η επιστρεφόμενη τιμή, όταν καλείται το σενάριο είναι ένα αλφαριθμητικό που περιέχει το όνομα του προπλάσματος. Εάν η πιο πάνω επιλογή δεν μεταβλήθηκε το αλφαριθμητικό θα περιέχει "Maple Leaves".

SF-GRADIENT

Θα δημιουργήσει μια ζώνη επιλογής στο διάλογο ελέγχου. Η ζώνη επιλογής αποτελείται από ένα κουμπί που περιέχει μια προεπισκόπηση της επιλεγμένης διαβάθμισης.

Εάν το κουμπί πιεστεί ένας διάλογος επιλογής διαβάθμισης θα αναδυθεί.

SF-GRADIENT "Gradient" "Deep Sea"

Η επιστρεφόμενη τιμή όταν το σενάριο καλείται είναι ένα αλφαριθμητικό που περιέχει το όνομα της διαβάθμισης. Εάν η πιο πάνω επιλογή δεν αλλάχτηκε το αλφαριθμητικό θα περιέχει "Deep Sea".

SF-PALETTE

Θα δημιουργήσει μια ζώνη επιλογής στο διάλογο ελέγχου. Η ζώνη επιλογής αποτελείται από ένα κουμπί που περιέχει το όνομα της επιλεγμένης παλέτας.

Εάν το κουμπί πιεστεί ένας διάλογος επιλογής παλέτας θα αναδυθεί.

SF-PALETTE "Palette" "Named Colors"

Η επιστρεφόμενη τιμή όταν το σενάριο καλείται είναι ένα αλφαριθμητικό που περιέχει το όνομα της παλέτας. Εάν η πιο πάνω επιλογή δεν αλλάχτηκε το αλφαριθμητικό θα περιέχει "Named Colors".

SF-FILENAME

Θα δημιουργήσει μια ζώνη επιλογής στο διάλογο ελέγχου. Η ζώνη επιλογής αποτελείται από ένα κουμπί που περιέχει το όνομα ενός αρχείου.

Εάν το κουμπί πιεστεί ένας διάλογος επιλογής αρχείου θα αναδυθεί.

SF-FILENAME "label" (string-append "" gimp-data-directory "/scripts/beavis.jpg")

Η επιστρεφόμενη τιμή όταν το σενάριο καλείται είναι ένα αλφαριθμητικό που περιέχει το όνομα του αρχείου.

SF-DIRNAME

Χρήσιμο μόνο στη διαδραστική κατάσταση. Παρόμοιο με το SF-FILENAME, αλλά η δημιουργούμενη ζώνη επιλογής επιτρέπει να επιλεγεί ένας κατάλογος αντί για αρχείο.

SF-DIRNAME "label" "/var/tmp/images"

Η επιστρεφόμενη τιμή όταν το σενάριο καλείται είναι ένα αλφαριθμητικό που περιέχει το όνομα του φακέλου.

SF-OPTION

Θα δημιουργήσει μια ζώνη επιλογής στο διάλογο ελέγχου. Η ζώνη επιλογής είναι ένα σύνθετο πλαίσιο που δείχνει τις επιλογές που πέρασαν ως λίστα.

Η πρώτη επιλογή είναι η προκαθορισμένη επιλογή.

SF-OPTION "label" '("option1" "option2")

Η επιστρεφόμενη τιμή όταν το σενάριο καλείται είναι ο αριθμός των επιλεγμένων επιλογών, όπου η πρώτη επιλογή ξεκινά από 0.

SF-ENUM

Θα δημιουργήσει μια ζώνη επιλογής στο διάλογο ελέγχου. Η ζώνη επιλογής είναι ένα σύνθετο πλαίσιο που δείχνει όλες τις απαριθμημένες τιμές για το δοσμένο τύπο απαρίθμησης. Αυτός πρέπει να είναι το όνομα της καταχωρισμένης απαρίθμησης χωρίς το πρόθεμα "Gimp". Η δεύτερη παράμετρος ορίζει την προκαθορισμένη τιμή, χρησιμοποιώντας το παρατσούκλι της τιμής απαρίθμησης.

SF-ENUM "Interpolation" '("InterpolationType" "linear")

Η επιστρεφόμενη τιμή όταν καλείται το σενάριο αντιστοιχεί στη επιλεγμένη τιμή απαρίθμησης.
